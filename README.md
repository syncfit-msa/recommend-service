# 📂 Commit 규칙

**형식**: [작업 유형] 작업 내용  

### 작업 유형:
- feat: 새로운 기능 추가
- fix: 버그 수정
- env: 개발 환경 관련 설정
- style: 스타일 수정 (코드 형식, 세미콜론 누락 등)
- refactor: 코드 리팩토링
- docs: 문서 추가 및 수정
- chore: 기타 작업 및 빌드 관련 코드 수정
- test: 테스트 작업

### 예시
```bash
git commit -m "feat: 이력서 기본 레이아웃 작성"
git commit -m "fix: CSS 스타일 오류 수정"
```
